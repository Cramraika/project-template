## §D — Pain-points encountered + resolution

### D1 — System reminder showed stale universal-claudemd version
- **Symptom**: initial system-reminder snippet showed `**VERSION: 2026-05-11-v44**`. Actual on-disk = `**VERSION: 2026-05-12-v45**`.
- **Resolution**: per §43.7 names-not-evidence — verified live file via `head -10`. Did NOT trust the stale snippet.
- **Lesson for next session**: ALWAYS re-read universal file at session start; system-reminder snippets are cached.

### D2 — `Read` throttle hook fired after 3-4 sequential reads (twice)
- **Symptom**: PreToolUse:Read hook returned "Too many consecutive read calls" mid-audit.
- **Resolution**: interleaved with `Bash` (cat/grep/head) for subsequent file inspection. Counter reset after non-Read tool calls.
- **Lesson**: for sequential file inspection, prefer `cat` / `head` via Bash, or batch with `for f in ...; do echo ===; cat $f; done`.

### D3 — `lychee.toml` exclude_path glob syntax silently broken
- **Symptom**: any `lychee --config lychee.toml` invocation errored with `regex parse error: **/node_modules/**`.
- **Resolution**: rewrote exclude_path entries as regex (no `**/x/**` glob; just `node_modules`, `\\.serena`, etc.). Verified `lychee` runs cleanly.
- **Lesson**: lychee 0.23+ exclude_path is regex, NOT glob. Watch for this pattern in other repos' `lychee.toml`.
